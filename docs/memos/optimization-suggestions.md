## 二、Novel 优化方向（让老板眼前一亮）

### 🔥 Priority 1：Interview Completion Rate Predictor（访谈完成率预测器）

**为什么 Novel？**
- 目前行业里没有任何一个竞品在做"实时预测访谈完成率"这件事
- 所有竞品都在优化"问什么"，没有人在优化"能不能聊完"
- 这是 **Quria 独有的数据优势**——只有你真正做了大量 AI 访谈，才有数据做预测

**核心逻辑：**
```
输入：回答长度变化趋势 + 回答间隔时间 + 追问后的回答质量 + 已访谈时长
  ↓
模型：轻量级预测（甚至用规则引擎先 demo）
  ↓
输出：该受访者"可能中途退出"的概率 → 触发自适应策略（缩短追问 / 提供激励话术）
```

**与 Project 1 的关系：**
Project 1 优化的是"追问本身的长度"，这个优化是"基于用户疲劳信号的**实时自适应**"。把两个项目串起来，讲一个完整的 story：**"我发现追问太长 → 我设计了动态长度规则 → 但我又想到，光有规则不够，需要根据用户实时状态自适应"**。

**技术边界话术：** "目前 demo 用的是规则引擎模拟，真实场景需要接入访谈日志数据训练预测模型，我设计好了数据采集 schema。"

---

### 🔥 Priority 2：Interview Quality Score（访谈质量评分系统）

**为什么 Novel？**
- 传统研究中，访谈质量全靠研究员经验判断，没有量化标准
- 如果能给每次访谈打一个"质量分"，就能自动筛选低质量访谈、评估 AI 访谈效果
- 这对客户交付非常有价值——"我们交付的不只是访谈记录，而是经过质量筛选的洞察"

**评分维度设计：**
| 维度 | 指标 | 数据来源 |
|------|------|----------|
| **深度性** | 追问轮次、回答信息密度 | 访谈日志 |
| **覆盖性** | 研究目标达成率 | 目标-回答匹配 |
| **自然度** | 对话流畅度、话题跳转合理性 | AI 评估 |
| **真实性** | 回答一致性、矛盾检测 | 交叉验证 |

**与 Project 3 的关系：**
Project 3 负责把客户需求翻译成技术方案，这个评分系统是"回传端"——**评估执行结果是否达标**。形成闭环：需求拆解 → 执行 → 质量评估 → 反馈优化。

---

### 🔥 Priority 3：Cross-Interview Pattern Discovery（跨访谈模式发现）

**为什么 Novel？**
- 单次访谈的价值有限，真正的洞察来自跨访谈的模式发现
- 国际竞品（Dovetail/Looppanel）在做的是"事后分析"，如果 Quria 能做**"访谈中实时模式发现"**，就是质的飞跃
- "第 23 个受访者说的关键词和第 7 个一样" → 实时标记给研究员

**Demo 思路：**
- 模拟 5-10 段访谈文本，展示关键词频率、主题聚类、情绪分布
- 可视化：词云 + 情感折线图 + 主题分布雷达图

---

### 💡 Priority 4：Adaptive Interview Flow（自适应访谈路径）

**为什么 Novel？**
- 现有的 AI 访谈（包括 Outset、Perspective AI）基本都是线性追问
- 如果能根据受访者的回答**动态调整访谈路径**（跳过已回答的、深入感兴趣的），体验会完全不同
- 类似"游戏分支剧情"的访谈体验

**与现有 Project 的关系：**
这是 Project 1（追问优化）的进化版——不只是优化追问长度，而是优化**整个访谈流程**。
