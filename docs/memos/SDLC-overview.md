# SDLC Overview（软件开发生命周期概述）

> **作者**: Hannah  
> **日期**: 2026-06-25  
> **类型**: Overview Memo  

SDLC 是 **Software Development Life Cycle**（软件开发生命周期）的缩写，简单来说就是：**一个软件从"想到要做什么"到"做出来、用起来、维护好"的整个过程。**

## 核心阶段

```
 ① 规划        ② 需求分析       ③ 设计
 (Planning)     (Requirements)   (Design)
      ↓              ↓               ↓
 ⑥ 维护        ⑤ 测试          ④ 编码/实现
(Maintenance)  (Testing)        (Implementation)
```

| 阶段 | 做什么 | 研究类比 |
|------|--------|----------|
| **① 规划** | 确定项目目标、范围、可行性 | 确定研究问题和范围 |
| **② 需求分析** | 弄清楚"用户到底需要什么" | 文献综述 + 研究设计 |
| **③ 设计** | 设计架构、界面、数据库等 | 理论框架设计 |
| **④ 实现/编码** | 写代码，把设计变成实际软件 | 收集数据 / 实验执行 |
| **⑤ 测试** | 检查有没有 bug，是否满足需求 | 验证研究结果 |
| **⑥ 维护** | 修复问题，更新迭代 | 后续研究与发表 |

## 常见的 SDLC 模型

不是所有项目都走同样的流程，常见模型有：

- **瀑布模型（Waterfall）**：一步一步来，走完上一步再做下一步。适合需求明确的项目。
- **敏捷开发（Agile）**：分成很多小轮次（sprint），每轮都产出可用的东西，不断调整。适合需求不确定的项目。
- **螺旋模型（Spiral）**：反复循环，每次循环都做一轮设计→实现→测试，同时评估风险。

## 跟研究有什么关系？

作为研究者，了解 SDLC 的意义在于：

1. **如果你研究的是软件/技术相关课题**——你需要知道你研究的对象（软件）是怎么被造出来的，才能理解它的设计决策、局限性、质量问题的根源。
2. **如果你自己要开发工具/原型**——SDLC 帮你组织开发过程，让你的工作更有条理，也更容易向别人解释"你是怎么做的"。
3. **方法论参考**——SDLC 本身就是一个人工制品（artifact），你可以研究不同 SDLC 模型对项目成果的影响，这也是一个常见的研究方向。

## 一句话总结

SDLC 就是软件的"一生"——从规划到退役的全过程框架。不需要精通每个细节，但理解它的基本流程，能帮助你在研究中更好地定位自己工作的上下文。
