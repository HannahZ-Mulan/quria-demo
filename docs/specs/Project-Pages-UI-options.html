Project Pages · UI Spec · 配套方案 B

让 Project 1 & 3 页面
交互、便捷、智能、高级

首页已是方案 B 深空科技风，但点进 project1（AI 追问）和 project3（需求拆解）后回到灰白平淡界面，体验断层。下面 3 个方向都沿用深空 + 霓虹 + 浮动，区别在「交互语言」：控制台式 / 对话流式 / 卡片工作台式。

💡 每个方案下方都有 真实可交互的页面演示——点按钮、切 Tab、hover 卡片都能看到效果。
方案 1

深空控制台 · Cockpit

把页面变成一块深色控制台仪表盘：左侧输入与参数区，右侧实时结果面板。结果用大号数字仪表呈现（字数 / 压缩率 / 策略），生成时按钮流光、结果弹出动画。最贴近方案 B 的「科技感」，适合数据型工具（project1 的字数优化）。

交互语言：仪表盘 / 实时数据 核心动效：按钮流光 + 数字仪表 + 结果弹出 改造量：中（重排布局 + 新增仪表组件） 最配：project1 字数优化

方案 2

智能对话流 · Conversational Flow

把整个流程变成一段引导式对话：顶部步骤指示器，主体是对话气泡，AI 主动提问 / 给建议，下方快捷操作 chip（一键精简 / 换模式 / 生成 TRD）。「打字中」三点动画 + 气泡逐条滑入。最体现「智能感」，适合 project3 这类多步骤拆解。

交互语言：对话 / 引导式 核心动效：步骤进度 + 气泡滑入 + 打字动画 + 快捷 chip 改造量：大（重构为对话形态） 最配：project3 需求拆解

方案 3

卡片工作台 · Workspace

保留原有表单结构，但用悬浮卡片网格 + 霓虹边框 + 流程节点重新包装。每个区块是一张可 hover 浮起的玻璃卡片（复用方案 B 的 TiltCard 语言），底部用发光流程节点串起步骤。改动最轻、与首页最统一，「成套感」最强，两个页面通用。

交互语言：卡片网格 / 模块化 核心动效：TiltCard 浮起 + 霓虹边 + 流程节点 + 结果 chip 改造量：小（替换容器 + 加动效，逻辑不动） 最配：两页通用、成套统一
