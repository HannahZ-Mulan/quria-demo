# SRS：AI 追问长度动态优化系统

## 1. 引言

### 1.1 目的
本文档为悦数洞察 Quria AI 访谈系统的「追问长度动态优化模块」提供详细软件需求规格说明，供开发、测试及验收使用。

### 1.2 范围
本模块覆盖：追问长度动态计算、问题语义压缩重构、研究员策略配置、效果数据追踪四个核心子系统。

### 1.3 定义与缩写

| 缩写 | 全称 |
|-----|------|
| Quria | 悦数洞察 AI 访谈平台 |
| 长度引擎 | 追问长度感知与计算引擎 |
| 重构模型 | 问题语义压缩重构模型 |
| 策略中心 | 研究员策略配置中心 |

---

## 2. 总体描述

### 2.1 产品视角
本模块作为 Quria AI 访谈引擎的增强组件，嵌入在现有追问生成流程中，不改变原有访谈架构。

```
[受访者回答] → [长度引擎] → [追问意图识别] → [重构模型] → [长度校验] → [输出追问]
                    ↑___________________________________________↓
                              [策略中心配置]
```

### 2.2 用户类型

| 用户类型 | 技术能力 | 使用场景 |
|---------|---------|---------|
| 受访者 | 无要求 | 参与 AI 访谈，接收优化后的追问 |
| 研究员 | 中等 | 配置访谈方案的长度策略 |
| 产品经理 | 中等 | 查看效果追踪看板 |
| 系统管理员 | 高 | 维护策略模板和阈值参数 |

### 2.3 运行环境
- 服务端：Python 3.10+，与现有 Quria 后端服务同部署
- 客户端：Web（PC/移动端自适应），兼容现有 Quria 前端框架

---

## 3. 功能需求

### 3.1 长度感知引擎（LE）

#### LE-FR-001：回答长度识别
**描述**：系统应能自动识别受访者上一轮回答的字数。  
**输入**：受访者文本回答  
**输出**：回答字数（整数）  
**处理**：去除标点、空格后计算中文字符数（英文按单词计）

#### LE-FR-002：动态长度计算
**描述**：系统应根据回答长度、当前话题复杂度、设备类型计算本轮追问建议字数。  
**规则**：

| 条件 | 建议字数上限 |
|-----|------------|
| 回答 ≤ 20 字 | 30 字 |
| 20 < 回答 ≤ 100 字 | 45 字 |
| 回答 > 100 字 | 60 字 |
| 设备类型 = 移动端 | 在上述基础上再降 25%（向下取整） |

#### LE-FR-003：话题复杂度评估
**描述**：系统应能评估当前话题的复杂度等级（简单/中等/复杂）。  
**输入**：当前访谈主题、已进行轮次、历史追问深度  
**输出**：复杂度等级（1-3）  
**处理**：基于预设话题标签库 + 历史数据模型推断

---

### 3.2 问题重构模型（RM）

#### RM-FR-001：语义压缩
**描述**：当生成的追问超过建议字数上限时，系统应调用重构模型进行语义压缩。  
**输入**：原始追问文本、追问意图标签、建议字数上限  
**输出**：压缩后追问文本  
**约束**：
- 不得改变追问意图标签
- 关键术语/专有名词可标记为例外，允许超限
- 压缩后语句通顺，符合中文表达习惯

#### RM-FR-002：追问拆分
**描述**：当追问内容无法在字数限制内完整表达时，系统应拆分为多个递进式短问题。  
**输入**：原始追问文本、建议字数上限  
**输出**：追问列表（1-3 个）  
**约束**：
- 拆分后的问题逻辑递进，不得跳跃
- 每个子问题 ≤ 建议字数上限
- 拆分点优先选择逻辑停顿处（如逗号、分号）

#### RM-FR-003：意图一致性校验
**描述**：压缩/拆分后，系统应进行意图一致性校验。  
**输入**：原始追问、优化后追问  
**输出**：一致性评分（0-100）  
**阈值**：≥ 90 分通过，< 90 分触发人工复核标记

---

### 3.3 策略配置中心（PC）

#### PC-FR-001：模式选择
**描述**：研究员应在研究方案配置页选择追问长度模式。  
**选项**：
- **精简模式**：短问题优先，适合快速筛查、移动端场景
- **标准模式**：动态平衡，默认推荐
- **深度模式**：允许较长追问，适合深度访谈场景

#### PC-FR-002：节点级覆盖
**描述**：研究员应能针对特定访谈节点单独覆盖长度限制。  
**输入**：节点 ID、自定义字数上限（10-100）  
**约束**：覆盖仅对当前节点生效，不影响全局模式

#### PC-FR-003：实时生效
**描述**：模式切换或节点覆盖后，已进行中的访谈应在下一轮追问即时生效。  
**约束**：不得中断当前轮次，不得要求受访者重新开始

---

### 3.4 效果追踪看板（DB）

#### DB-FR-001：核心指标展示
**描述**：看板应展示以下核心指标：
- 平均追问字数（按日/周/月）
- 访谈完成率
- 平均访谈时长
- 受访者满意度评分（如有问卷数据）

#### DB-FR-002：多维度下钻
**描述**：支持按以下维度筛选和下钻：
- 追问长度模式（精简/标准/深度）
- 行业标签
- 受访者年龄段
- 设备类型

#### DB-FR-003：A/B 测试对比
**描述**：支持创建 A/B 测试实验，对比不同长度策略的效果。  
**输入**：实验名称、对照组（原策略）、实验组（新策略）、样本比例  
**输出**：实验报告（完成率、时长、满意度对比）

#### DB-FR-004：自动预警
**描述**：当某模式下访谈完成率较基线下降 > 5% 时，系统应自动发送预警通知。  
**通知方式**：邮件 + 站内消息  
**接收人**：产品经理、相关研究员

## 4. 非功能需求

### 4.1 性能需求

| 编号 | 需求 | 指标 |
|-----|------|------|
| NF-001 | 追问生成延迟 | ≤ 500ms（含长度优化流程） |
| NF-002 | 重构模型响应时间 | ≤ 200ms |
| NF-003 | 看板数据刷新 | ≤ 5 分钟延迟 |
| NF-004 | 系统并发支持 | 不低于现有 Quria 标准 |

### 4.2 可用性需求

| 编号 | 需求 |
|-----|------|
| NF-005 | 策略配置页操作步骤 ≤ 3 步完成模式切换 |
| NF-006 | 看板关键指标默认展示，无需额外配置 |
| NF-007 | 所有错误提示使用中文，避免技术术语 |

### 4.3 安全需求

| 编号 | 需求 |
|-----|------|
| NF-008 | 研究员节点覆盖配置需记录操作日志 |
| NF-009 | A/B 测试实验数据隔离，防止交叉污染 |
| NF-010 | 效果数据访问权限按角色控制 |

### 4.4 兼容性需求

| 编号 | 需求 |
|-----|------|
| NF-011 | 与现有 Quria 访谈引擎 API 兼容，不破坏原有接口 |
| NF-012 | 支持渐进式上线（灰度发布），可快速回滚 |

---

## 5. 接口需求

### 5.1 内部接口

| 接口 | 方法 | 输入 | 输出 |
|-----|------|------|------|
| `/api/v1/length/calculate` | POST | 回答文本、话题 ID、设备类型 | 建议字数上限、复杂度等级 |
| `/api/v1/reconstruct/compress` | POST | 原始追问、意图标签、字数上限 | 压缩后追问、一致性评分 |
| `/api/v1/reconstruct/split` | POST | 原始追问、字数上限 | 追问列表 |
| `/api/v1/strategy/get` | GET | 研究方案 ID | 当前模式、节点覆盖配置 |
| `/api/v1/strategy/update` | POST | 研究方案 ID、模式、覆盖配置 | 更新结果 |

### 5.2 外部接口
- 复用现有 Quria 用户认证、日志、通知系统

---

## 6. 数据需求

### 6.1 数据存储

| 数据类型 | 存储方式 | 保留周期 |
|---------|---------|---------|
| 追问长度优化日志 | 关系型数据库 | 1 年 |
| 重构前后文本对比 | 对象存储 | 6 个月 |
| A/B 测试实验数据 | 关系型数据库 | 2 年 |
| 效果追踪指标 | 时序数据库 | 1 年 |

### 6.2 数据质量
- 所有追问文本需记录原始版本和优化版本
- 一致性评分 < 90 的数据需标记待复核
- 策略变更操作需记录变更前后值

---

## 7. 验收标准汇总

| 需求编号 | 验收标准 | 验证方式 |
|---------|---------|---------|
| LE-FR-002 | 动态长度计算符合规则表 | 单元测试 |
| RM-FR-001 | 压缩后追问字数 ≤ 建议上限 | 自动化测试 |
| RM-FR-003 | 意图一致性 ≥ 90% | 人工抽检（100 条/周） |
| PC-FR-001 | 三种模式可正常切换 | 功能测试 |
| PC-FR-003 | 模式切换后下轮即时生效 | 集成测试 |
| DB-FR-004 | 完成率下降 > 5% 触发预警 | 模拟测试 |
| NF-001 | 追问生成延迟 ≤ 500ms | 性能测试 |
